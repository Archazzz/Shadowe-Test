## 01_全局设计系统

### 1.1 色彩方案

以**紫藤色（Wisteria）** `#8B6CF6` 为品牌主色调，全面取代v1.0的蓝绿色系。淡紫色在色彩心理学中传达创新、亲和力与专业感，更适合AI助手类产品的定位。

| 角色 | 色值 | 用途 |
|------|------|------|
| **主色 Primary** | `#8B6CF6` (紫藤色) | 主按钮、选中状态、导航栏图标高亮、进度条 |
| **主色深色** | `#7C5CFC` | 重要强调、高亮标签、选中态加深 |
| **主色浅色** | `#B49DFB` | 次级按钮、hover状态、渐变终点 |
| **主色淡色** | `#D4C8FD` | 边框、分割线、浅色装饰 |
| **主色极淡** | `#EDE8FE` | 页面背景、卡片底色、标签背景、hover底色 |
| **主色渐变** | `linear-gradient(135deg, #8B6CF6 0%, #B49DFB 100%)` | 品牌区域、录音按钮、关键CTA背景 |
| **背景色** | `#F7F5FF` | 全局页面背景，极淡紫色调营造温暖氛围 |
| **强调色 Accent (CTA)** | `#FF6B9D` (珊瑚粉) | CTA按钮、重要状态提示、Badge，与主色形成互补对比 |
| **辅助色渐变** | `linear-gradient(135deg, #FF6B9D, #FFB8D1)` | 导出按钮、升级提示、强调操作 |
| **成功色 Success** | `#34C759` | 完成状态、成功提示 |
| **警告色 Warning** | `#FF9500` | 等待状态、警告提示 |
| **错误色 Danger** | `#FF3B30` | 删除确认、错误提示 |
| **卡片背景** | `#FFFFFF` + `backdrop-filter: blur(12px)` | 毛玻璃卡片 |
| **卡片边框** | `1px solid #EDE8FE` | 卡片描边，增加层次感 |
| **卡片阴影** | `0 4px 20px rgba(139, 108, 246, 0.08)` | 卡片悬浮阴影，带品牌紫色调 |
| **彩色阴影** | `0 8px 24px rgba(139, 108, 246, 0.15)` | 主按钮、重要卡片阴影 |
| **文字主色** | `#1A1A2E` | 大标题、页面标题（非纯黑，降低刺眼感） |
| **文字正文** | `#333344` | 主要内容文字、描述 |
| **文字次色** | `#666677` | 辅助说明、时间戳、提示信息 |
| **文字占位** | `#9999AA` | 输入框placeholder、禁用状态 |
| **分割线** | `#E8E8F0` | 列表分割线、卡片内分割 |

### 1.2 字体规范

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| **Display** | 32px | 700 | 1.2 | 启动页品牌名 |
| **H1** | 22px | 600 | 30px | 页面顶部标题、重要模块标题 |
| **H2** | 18px | 600 | 26px | 区块标题、卡片标题 |
| **H3** | 16px | 500 | 24px | 列表标题、子区块 |
| **Body** | 14px | 400 | 22px | 正文内容 |

字体栈：`-apple-system, BlinkMacSystemFont, 'PingFang SC', 'SF Pro SC', 'Helvetica Neue', 'Microsoft YaHei', sans-serif`

### 1.3 圆角与间距

**圆角层级：**

| 元素 | 圆角值 | 说明 |
|------|--------|------|
| 页面卡片 | `16px` | 大卡片，营造悬浮感和呼吸感 |
| 按钮（默认） | `12px` | 略大圆角增强亲和力 |
| 按钮（胶囊） | `9999px`（全圆角） | 全宽胶囊按钮、搜索框 |
| 输入框 | `12px` | 圆润的输入体验 |
| 标签/Tag | `8px` | 适中圆角 |
| 头像 | `50%` | 圆形 |
| 列表项 | `0` | 直角，底部分割线 |
| 弹窗 | `16px` | 大圆角降低压迫感 |
| 底部浮层 | `16px` (仅顶部) | 顶部圆角，底部贴边 |

**间距层级：**

| 间距层级 | 值 | 用途 |
|----------|-----|------|
| 页面边距 | `16px` | 左右安全区 |
| 区块间距 | `24px` | 主要区块之间 |
| 元素间距 | `16px` | 相关元素之间、卡片内边距 |
| 紧凑间距 | `12px` | 卡片之间间距 |
| 微观间距 | `8px` | 图标与文字之间 |
| 极紧凑 | `4px` | 紧密元素间距 |

### 1.5 阴影规范

| 层级 | 阴影值 | 使用场景 |
|------|--------|----------|
| 微阴影 | `0 1px 3px rgba(139, 108, 246, 0.06)` | 标签、小型按钮hover |
| 小阴影 | `0 2px 8px rgba(139, 108, 246, 0.08)` | 卡片、列表项 |
| 中阴影 | `0 4px 16px rgba(139, 108, 246, 0.10)` | 下拉菜单、气泡提示、悬浮按钮 |
| 大阴影 | `0 8px 24px rgba(139, 108, 246, 0.12)` | 弹窗、模态框、重要浮层 |
| 彩色阴影 | `0 8px 24px rgba(139, 108, 246, 0.20)` | 主按钮、重要操作强调 |

> **原则**：所有阴影使用带紫色调的rgba值，避免纯黑`rgba(0,0,0,x)`的冷硬感。

### 1.6 动效规范

| 动效类型 | 时长 | 缓动函数 | 用途 |
|----------|------|----------|------|
| 页面进入 | `300ms` | `cubic-bezier(0.4, 0, 0.2, 1)` | 页面切换 |
| 按钮点击 | `150ms` | `ease-in-out` | 缩放至0.98 |
| 卡片悬浮 | `200ms` | `ease` | 阴影加深+微上移`translateY(-2px)` |
| 录音波形 | 实时 | `linear` | 波形条高度变化 |
| 进度条 | `800ms` | `ease-in-out` | 平滑过渡 |
| 骨架屏 | `1.5s` | `linear` | shimmer 闪烁 |
| 徽标呼吸 | `2s` | `ease-in-out` | 无限循环缩放 |

#### 关键交互

| 交互元素 | 事件 | 响应 |
|----------|------|------|
| CSV/Excel上传卡片 | `tap` | 调用`wx.chooseMessageFile({ type: 'file', extension: ['csv','xlsx','xls'] })` |
| 粘贴数据卡片 | `tap` | 弹出底部文本输入浮层 |

#### 数据流

```
输入:
  - projectId: query参数（可选，如从项目页进入则自动关联）
  - questionnaire_file: CSV/Excel文件
  - strategy: 'deep_insight' | 'pain_mining' | 'typical_user' | 'high_value'

步骤1 - 导入流程:
  1. 用户选择文件 → 前端CSV解析（papaparse库）
  2. 前端校验: 文件大小(≤10MB), 格式(.csv/.xlsx), 行数(≥10行)
  3. 字段自动识别: 检测表头 → 分类为人口统计/量表题/开放题/筛选题
  4. 数据质量检测: 空值率、重复行、极端值
  5. 用户确认字段映射 → POST /api/questionnaires/import
     Body: { projectId, fields[], responses[], qualityReport }
  6. 响应: { importId, fieldCount, responseCount, qualityScore }

步骤2 - AI筛选流程:
  POST /api/questionnaires/{importId}/screen
  Body: { strategy, weights? }
  
  筛选策略权重配置:
  - deep_insight:   { answerQuality: 0.30, expressiveness: 0.20, extremity: 0.20, roleMatch: 0.15, reliability: 0.15 }
  - pain_mining:    { extremity: 0.30, answerQuality: 0.25, strategic: 0.20, roleMatch: 0.15, expressiveness: 0.10 }
  - typical_user:   { roleMatch: 0.30, consistency: 0.25, reliability: 0.20, answerQuality: 0.15, expressiveness: 0.10 }
  - high_value:     { strategic: 0.30, roleMatch: 0.25, answerQuality: 0.20, expressiveness: 0.15, reliability: 0.10 }

  响应: { 
    candidates: [{ 
      userId, score, rank, 
      dimensionScores: { answerQuality, extremity, roleMatch, expressiveness, reliability, strategic },
      keyReason, aiConfidence
    }],
    summary: { totalScreened, selectedCount, avgScore, strategy }
  }

步骤3 - 生成访谈问题:
  POST /api/questionnaires/{importId}/generate-questions
  Body: { candidateIds[], researchGoal }
  
  响应: {
    guides: [{
      candidateId,
      questions: [{ type, source, question, followUps[] }]
    }]
  }

问题类型枚举:
  - 'behavioral': 行为描述题
  - 'probing': 梯级探询题
  - 'pain_recall': 困难回顾题
  - 'scenario': 情景反应题
  - 'attitude': 态度量表题

输出:
  - POST /api/questionnaires/{importId}/save-guides
  - Body: { guides[], projectId }
  - 响应成功后 → 跳转项目详情页
```
